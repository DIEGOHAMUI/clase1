Areas de GIT

Working Directory

El area de trabajo corresponde a la carpeta donde inicializamos el repositorio de git y realizamos la creación, edición y eliminación de archivos en nuestro proyecto. En esta instancia haremos muy poco uso de la lina de comandos, mas alla de algunas instrucciones basicas.

  • dir \\ ls muestra el contenido del directorio actual
  • mkdir < name> crea una carpeta con el nombre especificado
  • cd < path> cambia el directorio actual por el seleccionado
  • git init iniciaiza un repositorio de git.
  • rm -rf .git elimina el repositorio de manera forzada

Staging Area (INDEX)

El area de control de cambios corresponde a un archivo que registra elementos le estamos haciendo seguimiento dentro de nuestro proyecto. Cada vez que agregamos un archivo al INDEX se realiza una captura del codigo(SNAPSHOT) que registra los cambios realizados; una vez realizada la misma, podemos descartarla o confirmarla.

  • git status muestra el estado de los archivos con respecto al INDEX, si se esta haciendo seguimiento de los mismos o no, y los posibles resultrados son los siguientes:
    • staged el archivo fue agregado al INDEX, pero no aun no se realizo la confirmación de cambios(cuando hice la captura)
    • commited los cambios del archivo fueron confirmados en un commit y pueden ser restablecidos en cualquier momento
    • modified el archivo difiere con respecto a la ultima confirmación realizada
  • git diff compara los cambios del Working Directory con respecto a los almacenados en el INDEX
  • git restore --staged <files> deshace la captura realizad en el INDEX
  • git rm --cached <file> quita el archivo seleccionado del area de control de cambios

Repository

El repositorio corresponde a la carpeta .git, donde se almacenan todos los cambios en formato BLOB(Binary large Objet File) utilizando referencias de los nombres implementados en el proyecto. Tambien se utiliza para navegar entre los commits como puntos de control

  • git commit realiza la confirmación de cambios de las capturas realizadas
  • git blame <file> muestra el contenido del archivo identificado al autor de cada cambio
  • git log muestra el historial de confirmación de forma descendente
  • git fetch <remote> <branch> camptura los cambios realizados en un repositorio remoto

Apuntadores GIT

HEAD

Apuntador Movil que se utiliza para indicar al usuario donde se encuentra actualmente, permite la navegación dentro del historial de cambios e incluso lo podemos utilizar como referente mediante expresiones como HEAD ^ o HEAD~1 para indicar a cuantos ancestros o commits anteriores hacemos referencia

  • git diff HEAD compara el contenido actual del working directory con el ultimo commit emitido
  • git checkout HEAD~1 mueve el apunador a la confirmación anterior a la que nos encontramos actualmente

BRANCH

Apuntador dinamico que siempre hace referencia al ultimo commit dentro de la historia en la que encontremos, se utilizan para implementar caracteristicas de prueba, realizar revisiones o correcciones, sin moficicar la linea principal del proyecto. Tambien faciitan la navegación dentro del historial de cambios.
